🧠 PROTOTIPO FUNCIONAL – Módulo de Inventario de Bioinstrumentación
✅ MÓDULO 1: Selección de Usuario
Interfaz inicial: Seleccionar tipo de usuario:

👨‍🎓 Estudiante

👩‍🏫 Docente

👩‍🔬 Laboratorista

Autenticación parcial:

Estudiante: solo selecciona su nombre o lo digita.

Docente / Laboratorista: ingresa con un código especial (tipo PIN o clave).

📦 MÓDULO 2: Panel de Acciones según Usuario
🎓 ESTUDIANTE
Puede:

Prestar elementos.

Retornar elementos.

Flujo:

Selecciona acción: Prestar o Retornar.

Selecciona categoría (ej. Arduino, Cables).

Selecciona elemento (ej. Arduino UNO R3).

Se muestra la imagen del elemento.

Selecciona cantidad.

Botón de acción: “Prestar” o “Retornar”.

Aparece un mensaje de confirmación y el inventario se actualiza en tiempo real.

Si la cantidad disponible es 0 → el botón se deshabilita.

👨‍🔬 LABORATORISTA
Puede:

Ver el inventario completo (restringido con código).

Prestar y retornar elementos como estudiante.

Agregar nuevos elementos.

Eliminar elementos.

Extra: módulo de consulta avanzada:

Filtros por categoría, nivel de inventario, histórico de movimientos.

👩‍🏫 DOCENTE
Igual al laboratorista, pero sin permiso para agregar o eliminar elementos.

📈 MÓDULO 3: Lógica de Inventario
Cada acción debe:

Disminuir cantidad si es préstamo.

Aumentar cantidad si es retorno.

Mostrar imagen referencial.

Registrar usuario, fecha, hora, acción y cantidad en una bitácora para trazabilidad.

📅 MÓDULO 4: Reportes (futuro)
(Opcional por ahora, pero fácil de integrar)

Informe mensual con:

Préstamos realizados.

Elementos no retornados.

Top 10 de materiales más prestados.

Stock crítico.

⚙️ SUGERENCIA TECNOLÓGICA PARA IMPLEMENTARLO
Como ya tienes experiencia con Laravel, propongo:

Backend: Laravel 10

Frontend: Livewire (para reactividad sin necesidad de usar Vue) o Inertia.js + React

Base de datos: MySQL o PostgreSQL

Auth: Laravel Breeze o Fortify para roles

Visualización: Bootstrap o Tailwind (Tailwind recomendado)

Y como puedes usar los computadores del laboratorio, se puede montar localmente en red interna o incluso dejarlo accesible desde la web (según necesidad).
